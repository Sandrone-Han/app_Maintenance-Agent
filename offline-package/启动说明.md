# 设备维护排班系统离线启动说明

本文件夹就是完整离线运行包。受限电脑不需要源码、Node.js 或 npm，只需要已经安装并启动 Docker Desktop。

## 文件说明

```txt
maintenance-scheduler-images.tar   Docker 镜像包，包含前端、后端和 Oracle
docker-compose.yml                 容器启动配置
.env.production                    端口和数据库配置
first-run.ps1                      第一次启动脚本
start.ps1                          日常启动脚本
stop.ps1                           停止脚本
status.ps1                         查看状态脚本
backend/db/init/                   Oracle 初始化挂载目录
```

## 第一次启动

在当前文件夹空白处按住 Shift，右键选择“在终端中打开”或“在 PowerShell 中打开”，执行：

```powershell
.\first-run.ps1
```

第一次启动会导入镜像、启动 Oracle、初始化数据库并启动系统，耗时会比日常启动更久。

启动完成后打开浏览器访问：

```txt
http://localhost:8080
```

## 以后启动

Docker Desktop 打开后，在当前文件夹执行：

```powershell
.\start.ps1
```

浏览器访问：

```txt
http://localhost:8080
```

## 停止程序

```powershell
.\stop.ps1
```

## 查看运行状态

```powershell
.\status.ps1
```

正常情况下会看到 `maintenance-frontend`、`maintenance-backend`、`maintenance-oracle` 三个容器。

## 常见问题

- 如果提示找不到 `docker`，说明 Docker Desktop 没安装或没启动。
- 如果网页打不开，先执行 `.\status.ps1`，确认三个容器都在运行。
- 如果其他电脑要访问这台电脑上的系统，使用 `http://这台电脑的IP:8080`，并确认防火墙允许 8080 端口。
- 不要删除 Docker volume `oracle-data`，否则数据库数据会丢失。
